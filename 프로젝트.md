# 🚗 자율주행 오토카(AutoCar) 시스템 구축 및 센서 퓨전 가이드

## 1. 프로젝트 개요
본 프로젝트는 NVIDIA Jetson 보드 환경에서 **라이다(LiDAR) 센서**와 **AI 비전 카메라**를 융합하여 자율주행 및 충돌 회피(Collision Avoidance) 시스템을 구축하는 것을 목표로 합니다. 백그라운드 환경에서 시스템을 안정적으로 구동하기 위해 `tmux` 기반의 4분할 엔진룸을 설계하였습니다.

## 2. 시스템 관제 환경 구축 (`tmux` & `jtop`)
자율주행 코드가 실행되는 동안 AI 모델이 젯슨 보드(GPU)에 미치는 부하를 실시간으로 모니터링하기 위해 전용 관제 시스템을 도입했습니다.

*   **`jtop` (jetson-stats) 설치:**
    가상환경 밖(Global)에서 아래 명령어로 설치하여 젯슨 보드의 GPU, CPU, 전력, 온도 상태를 모니터링합니다.
    ```bash
    sudo -H pip3 install -U jetson-stats
    ```
*   **4분할 관제 시스템:** `tmux`를 사용하여 모터 제어, AI 엔진, 통신 허브, 시스템 모니터링 창을 한 화면에 분할하여 실행합니다.

## 3. 코드 변환 및 트러블슈팅
백그라운드 터미널 구동을 위해 기존 주피터 노트북 파일(`.ipynb`)을 순수 파이썬 파일(`.py`)로 변환하는 작업을 진행했습니다.

*   **에러 원인:** 터미널에서 `.ipynb` 파일의 JSON 구조를 해석하지 못해 실행 에러 발생.
*   **해결 방법:** `nano` 편집기를 사용하여 찌꺼기 없는 순수 파이썬 텍스트 파일 생성.
    ```bash
    nano motor_fusion.py
    ```
*   **들여쓰기(Indentation) 에러 해결:** `nano` 편집기에 코드를 붙여넣을 때 발생하는 계단 현상(자동 들여쓰기 오류)을 백스페이스로 기준선에 맞춰 정렬하여 해결.

## 4. 자율주행 센서 퓨전 (Sensor Fusion) 핵심 로직
카메라 AI 모델(딥러닝)과 라이다 센서(규칙 기반)를 결합하여 이중 안전망을 구축했습니다.

*   **AI 비전 (카메라):** 300x300 해상도로 직접 수집한 600장(블록 O/X)의 데이터를 사전 학습(`CA.train(times=10)`)하여 시각적인 장애물을 인지합니다.
*   **물리 센서 (라이다):** 학습 없이 즉각적인 거리(700mm 이내) 데이터를 기반으로 물리적 방어막 역할을 수행합니다.
*   **융합 제어 조건:** 라이다가 70cm 이내의 장애물을 감지하거나, AI 모델이 시각적으로 블록을 인지했을 때 즉각 후진 및 회피 기동을 실시합니다.

## 5. 자동화 실행 스크립트 (`start_autocar.sh`)
모든 시스템을 한 번에 실행하고 관제 화면으로 진입하는 최종 쉘 스크립트입니다.

```bash
#!/bin/bash

SESSION_NAME="autocar"

tmux has-session -t $SESSION_NAME 2>/dev/null
if [ $? == 0 ]; then
    echo "세션이 이미 존재합니다. 연결합니다."
    tmux attach -t $SESSION_NAME
    exit 0
fi

tmux new-session -d -s $SESSION_NAME

# 화면 4분할
tmux split-window -h -p 50
tmux select-pane -t 0
tmux split-window -v -p 50
tmux select-pane -t 2
tmux split-window -v -p 50

# Pane 0: 카메라 & 레이더 센서 퓨전 모터 제어 (.py 실행)
tmux send-keys -t 0 'source venv/bin/activate' C-m
tmux send-keys -t 0 'python3 motor_fusion.py' C-m

# Pane 1: AI 엔진 (.py 실행)
tmux send-keys -t 1 'source venv/bin/activate' C-m
tmux send-keys -t 1 'python3 ai_engine.py' C-m

# Pane 2: 통신 허브 Flask 웹 서버 (.py 실행)
tmux send-keys -t 2 'source venv/bin/activate' C-m
tmux send-keys -t 2 'python3 app.py' C-m

# Pane 3: 시스템 하드웨어 모니터링 (jtop)
tmux send-keys -t 3 'jtop' C-m

# 완성된 세션으로 진입
tmux attach-session -t $SESSION_NAME
```

### 💡 주요 명령어 모음
*   **스크립트 실행:** `./start_autocar.sh`
*   **기존 세션 초기화(삭제):** `tmux kill-server`
*   **nano 편집기 저장 및 종료:** `Ctrl + O` (저장) ➡️ `Enter` ➡️ `Ctrl + X` (종료)

# 미니 과제

## 목표

- `pywebview`를 이용한 대시보드 만들기
- `autocar`에 API를 `Flask`로 구현해서 작동하게 만들기
- 모터 구동(조이스틱 구현)
- 카메라 pan/tilt 구현(슬라이더)
- 카메라 화면을 버튼으로 조절
- TTS를 사용해서 안내 음성 또는 멜로디 재생
- 프로그램은 `.py` 파일로 만든 뒤 `scp`로 이동시켜 터미널에서 실행

---

## 개요

Autocar 내부에서는 Python API 서버를 실행한다.
노트북에서는 `pywebview` 기반 대시보드를 실행한다.

노트북 대시보드에서 버튼, 조이스틱, 슬라이더를 조작하면 Autocar 내부 API로 명령이 전달된다.
Autocar는 전달받은 명령에 따라 모터, 조향 서보, 카메라 팬/틸트 서보, 스피커, 카메라 스트리밍을 제어한다.
카메라 화면은 노트북 `pywebview` 화면 안에서 볼 수 있어야 한다.

위험한 동작을 막기 위해 정지 버튼, 속도 제한, 입력값 제한을 반드시 구현한다.

## 구현해야 할 기능

### 1. Autocar 내부 API 서버

#### `/api/status`

- 현재 서버 상태, 카메라 상태, 마지막 명령, 현재 속도, 현재 조향값을 JSON으로 반환

#### `/api/motor`

- `POST` 방식
- 입력 예시:
  ```json
  {"speed": 30}
  ```
- `speed` 범위: `-100 ~ 100`
- 양수: 전진
- 음수: 후진
- `0`: 정지

#### `/api/steering`

- `POST` 방식
- 입력 예시:
  ```json
  {"angle": 0}
  ```
- `angle` 범위: `-45 ~ 45`
- 음수: 좌회전
- 양수: 우회전

#### `/api/drive`

- `POST` 방식
- 입력 예시:
  ```json
  {"x": 0.3, "y": 0.8}
  ```
- 조이스틱 입력을 받아 `x`는 조향, `y`는 속도로 변환
- `x`, `y` 범위: `-1.0 ~ 1.0`

#### `/api/camera/pan_tilt`

- `POST` 방식
- 입력 예시:
  ```json
  {"pan": 90, "tilt": 60}
  ```
- `pan`, `tilt` 범위는 실제 서보 범위에 맞게 제한
- 카메라 방향 조절

#### `/api/sound/tone`

- `POST` 방식
- 입력 예시:
  ```json
  {"freq": 440, "duration": 0.5}
  ```
- 지정한 주파수와 시간으로 소리 재생

#### `/api/sound/tts`

- `POST` 방식
- 입력 예시:
  ```json
  {"text": "hello autocar"}
  ```
- 인터넷 가능 시 `gTTS`
- 인터넷 불가 시 `espeak` 또는 `piper` 사용 가능하도록 설계

#### `/video_feed`

- OpenCV 카메라 영상을 MJPEG 스트리밍으로 제공
- 노트북 `pywebview`의 `img` 태그에서 볼 수 있어야 함

#### `/api/stop`

- `POST` 방식
- 모터 속도를 `0`으로 만들고 조향을 중앙으로 복귀
- 긴급 정지 버튼에서 호출

### 2. 노트북 pywebview 대시보드

#### `app.py`

- `pywebview` 창 생성
- `index.html` 로드
- Autocar API 서버 주소 설정 가능
- 예:
  - `http://192.168.55.1:8000`
  - WiFi IP 주소

#### `index.html`

- 카메라 화면 영역
- 조이스틱 영역
- 전진/후진/좌/우/정지 버튼
- 속도 슬라이더
- 카메라 pan/tilt 슬라이더
- 소리 재생 버튼
- TTS 입력창과 말하기 버튼
- 현재 상태 표시 영역

#### `main.js`

- `fetch` API로 Autocar API 서버에 명령 전송
- 조이스틱은 마우스 또는 터치 이벤트로 구현
- 조이스틱 값은 `-1.0 ~ 1.0` 사이로 정규화
- 일정 주기로 `/api/status` 호출하여 상태 표시
- 통신 실패 시 에러 메시지 표시

#### `style.css`

- 대시보드처럼 보이게 화면 구성
- 카메라 영역, 제어 패널, 상태 패널 구분
- 모바일/노트북 화면에서도 사용 가능하도록 반응형 작성

## 중요한 구현 조건

- 실제 하드웨어 제어 코드는 바로 작성하지 말고, 먼저 `Mock` 클래스로 동작하게 한다.
- `Mock` 클래스에서는 `print` 문으로 `motor`, `steering`, `pan`, `tilt`, `sound` 명령을 확인할 수 있어야 한다.
- 이후 실제 Autocar 라이브러리 또는 GPIO/서보 제어 코드로 교체할 수 있도록 클래스를 분리한다.
- API 서버와 대시보드는 각각 독립 실행 가능해야 한다.
- CORS 문제가 발생하지 않도록 API 서버에서 CORS 설정을 고려한다.
- 카메라가 없는 환경에서도 테스트할 수 있도록 더미 이미지 또는 `"camera not available"` 처리를 넣는다.
- 모든 API 입력값은 범위를 검사하고, 잘못된 값이면 에러 JSON을 반환한다.
- `README.md`에는 실행 방법, IP 설정 방법, API 목록, 테스트 방법을 작성한다.

1. 시스템 아키텍처 및 통합 관제 환경 구축
본 프로젝트의 핵심은 시스템의 안정성을 위해 RC카 내부에서 묵묵히 일하는 '백그라운드 엔진룸'과, 사용자가 눈으로 보고 직관적으로 조종하는 '프론트엔드 관제탑'을 완벽하게 분리하는 데 있습니다. 시스템 제어 및 개발은 기 구축된 주피터(Jupyter) 기반 웹 터미널 환경 위에서 진행됩니다.

1.1. RC카 내부 엔진룸 (tmux 백그라운드 구동)
Jupyter 터미널 안에서 자동화 스크립트(start_autocar.sh)를 실행하여 tmux의 4개 창(Pane)을 백그라운드 엔진으로 가동합니다. 웹 브라우저 탭이 닫히거나 네트워크가 끊기더라도, tmux가 자율주행 프로세스를 꽉 잡고 유지하므로 차량은 안전하게 주행합니다.

Pane 0 (모터 엔진 & 센서 퓨전): [핵심 제어] 카메라 영상과 레이더 센서 데이터를 융합(Sensor Fusion)합니다. 카메라로 읽은 차선 데이터를 기반으로 PID 조향을 수행하고, 동시에 레이더로 측정한 전방 거리 데이터를 바탕으로 장애물 회피, 긴급 제동(AEB), 주행 속도 동적 조절을 수행하는 메인 코드가 구동됩니다.

Pane 1 (AI 엔진): YOLO와 CNN 모델이 영상을 분석하여 표지판이나 특정 마커를 실시간으로 분류하고 인식합니다.

Pane 2 (통신 허브 / 웹 서버): Flask 기반의 웹 서버가 실행되며, 다른 Pane들의 데이터(영상, 레이더 거리값, 센서 로그)를 외부로 송출하고 모바일의 조종 명령을 수신하는 '우체국' 역할을 수행합니다.

Pane 3 (시스템 모니터): 하드웨어의 온도, CPU, RAM 상태를 실시간으로 측정하여 통신 허브로 보냅니다.

1.2. 사용자 맞춤형 프론트엔드 관제탑 (Client Web View)
RC카 내부의 통신 허브(Pane 2)는 접속하는 기기에 맞춰 두 가지의 완벽히 분리된 웹 화면(UI)을 제공합니다. (포트 5000번 사용)

📱 모바일 리모컨 뷰 (핸드폰 접속 시 - /remote):
핸드폰 핫스팟 네트워크를 활용해 접속합니다. 무거운 영상 스트리밍 없이 가상 조이스틱 UI만 띄워, 지연 없는 터치 제어와 비상정지 역할을 수행합니다.

💻 통합 대시보드 뷰 (노트북/모니터 접속 시 - /dashboard):
종합 관제 센터입니다. AI 라이브 영상(Vision Feed) 위에 레이더가 감지한 전방 거리 수치가 함께 오버레이(Overlay)되어 표시되며, 모바일 제어 로그와 시스템 부하 상태 그래프(System Metrics)가 단 하나의 웹 화면에 통합 출력됩니다.

1.3. 기초 소프트웨어 및 하드웨어 환경 세팅 (Environment Setup)
위의 환경이 에러 없이 원활하게 구동되기 위한 필수 기반 공사입니다.

격리된 가상환경: 주피터 커널과 연동되는 프로젝트 전용 파이썬 가상환경(venv)을 구동하여 패키지 충돌을 방지합니다.

필수 라이브러리: 영상 및 AI 처리를 위한 opencv-python, torch, 웹 통신을 위한 Flask, Flask-SocketIO와 함께 레이더 시리얼 통신을 위한 pyserial 패키지를 세팅합니다.

하드웨어 직통 권한: 리눅스 OS에서 프로그램이 관리자 권한 없이도 카메라 모듈과 레이더 시리얼 포트, 모터 제어 포트에 즉시 접근할 수 있도록 사용자 계정에 video, dialout 권한을 설정합니다.
